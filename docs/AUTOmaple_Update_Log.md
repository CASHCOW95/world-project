# AUTOmaple 프로젝트 관리 로그

## 1. 정식 버전 정보 (v1.0.0)
- **버전명**: v1.0.0 (Master Original)
- **배포 일자**: 2026.06.03
- **주요 사양**:
  - **Human-Engine**: 베지에 곡선 및 가속도 제어 기반의 인간미 있는 마우스 트래킹 알고리즘 적용.
  - **도형 추적 엔진**: `anti1.png` 기반 색상 자동 추출 및 배경 차분을 통한 동적 경로 추적.
  - **UI 시스템**: `create_data_row` 메서드를 통한 실시간 시스템 지표(CPU, RAM, 가동 시간 등) 대시보드 구현.
  - **안전성**: F2(시작), F3(종료) 단축키 및 비정상 종료 시 마우스 상태 강제 해제 로직 탑재.

## 2. 향후 업데이트 로드맵 (v1.0.1 예정)
- **핵심 기술 도입: Lucas-Kanade 광학 흐름(Optical Flow)**
  - 단순 색상 및 배경 차분을 넘어, 객체의 코너(Corner)나 특징점(Feature Points)을 추출하여 벡터 단위로 움직임을 추적하는 Lucas-Kanade 알고리즘을 전면 도입합니다.
  - 이를 통해 투명도가 매우 높거나 배경과 유사한 도형이 빠르게 튀는 상황(비선형/불규칙 경로)에서도 탈조 없는 완벽한 추적 정밀도를 달성할 계획입니다.

## 3. 버전 관리 원칙
사용자의 요청에 따라 다음과 같은 점진적 업데이트 체계를 유지합니다:
1. **버전 증분**: 모든 수정 및 업그레이드 시 버전 번호를 **0.0.1** 단위로 상향한다. (v1.0.0 -> v1.0.1 -> v1.0.2 ...)
2. **원본 보존**: v1.0.0은 정식 원본으로 정의하며, `deploy/backup/v1.0.0` 폴더에 소스 코드와 실행 파일을 영구 백업한다.
3. **업데이트 프로세스**: 
   - 사용자의 수정 요구사항 발생 시 코드를 수정한다.
   - 수정 완료 후 새 버전으로 실행 파일을 빌드하여 `deploy/dist`에 배포한다.
   - 변경 사항을 본 문서에 지속적으로 기록한다.

## 4. 업데이트 히스토리
- **v1.0.1 (2026.06.03)**: Lucas-Kanade 알고리즘 도입을 통한 트래킹 엔진 고도화 완료.
- **v1.0.2 (2026.06.03)**: 백그라운드 스레드 CPU 자원 최적화 적용. 사냥 비가동 시 탐지 루프 일시 정지, 거탐 팝업 탐지 시 도형 추적 자동 활성화(ON) 및 추적 유실 시 자동 해제(OFF) 연동, 유휴 상태 대기 시간(0.5초) 연장.
- **v1.0.3 (2026.06.03)**: 리소스명 개편 동기화 및 단축키 안내 라벨 적용. 배경 패턴으로 `anti0.png` 적용, 거탐 감지 인덱스 범위를 `range(1, 4)`로 한정하여 배경 매칭 오작동 차단, UI 상의 도형 추적 엔진 활성화 라벨에 F2/F3 단축키 표시 추가. 빌드 결과물 배포 경로를 `core/exe` 하위 폴더로 조정 완료.
- **v1.0.4 (2026.06.03)**: 미니게임 팝업 픽셀 영역 크롭(ROI) 및 마우스 궤적 오프셋 보정 기능 도입. `anti0.png` / `anti0.1.png`를 이용해 화면 내 팝업 윈도우의 정확한 상대 좌표를 검출하고, 해당 관심 영역(ROI)만 잘라내어 분석을 수행하도록 알고리즘을 개선함. 마우스 이동 경로 계산 시 크롭 시작점 좌표를 추가 보정하여 정밀 추적 기능 확립.
- **v1.0.5 (2026.06.03)**: 팝업 배경 매칭 임계값을 `0.7`에서 `0.5`로 하향 조정하여 팝업 미검출 현상 개선 및 자동 추적 안정화.
- **v1.0.6 (2026.06.03)**: 템플릿 매칭 실패 시 디바이스 해상도 기반의 중앙 ROI 검출 폴백(FHD 1920x1080일 때 매치 좌표 448, 205 강제 매핑 및 기타 해상도 중앙 정렬 수식 보정) 로직 구현으로 팝업 감지 강인성 확보.
- **v1.0.8 (2026.06.03)**: 점점 희미해지는 하얀 도형 검출 개선. 배경 차분 결과 이미지에 명암비 정규화(Contrast Normalization)를 적용하여 미세한 경계선을 극대화하고, Canny 임계치 하향 조정 및 윤곽선 solidity 제한 완화를 통해 극도로 흐려진 테두리 검출율을 확보함. 추가적으로, 타겟이 일시적으로 완전히 투명해져 검출이 실패하는 구간에서 이전 속도 벡터를 기반으로 마우스 드래그를 유지하며 계속 추적하는 관성 기반 예측 추적(Inertial Tracking) 로직을 탑재하여 트래킹 연속성을 보장함.
- **v1.0.9 (2026.06.03)**: 투명 도형 검출 기능 복원. 이전 버전의 명암비 정규화 기법이 미세 노이즈까지 극대화하여 검출을 방해하던 문제를 확인하여 배제함. 대신 배경 차분 이미지에 가우시안 블러(Gaussian Blur)와 저임계값(임계치 6) 이진화(Binarization) 필터를 도입하여 희미한 도형의 닫힌 윤곽선(Blob)을 안정적으로 검출하도록 전면 개편함. solidity 하한 임계값을 0.3으로, 검출 면적 범위를 400~40000으로 유연화하여 형상 왜곡에 대한 강인성 보완.
- **v1.1.0 (2026.06.03)**: 투명 도형 2단계 추적(초기 흰색 락온 + 로컬 ROI 테두리/관성 트래킹) 엔진 탑재. 도형 최초 생성 시 완전한 흰색(#ffffff) 계열이라는 특징을 이용해 Gray 240 이상에서 1단계 락온(Lock-on)을 수행하고, 락온 후 희미해지는 2단계에서는 이전 도형 중심 기준 160x160 크기의 로컬 윈도우 영역만 집중 연산(배경 차분 + Blur + Threshold 5 이진화)함으로써 주변 노이즈를 100% 차단하며 투명 테두리를 정밀 트래킹하도록 개조함. LK 특징점의 주기적 로컬 갱신 및 완전 유실 시 대기 복원 로직 추가.
- **v1.1.1 (2026.06.03)**: 트래킹 정밀화 및 마우스 튐 방지 패치. 초기 락온 시 도형의 정조준 보장을 위해 마우스 mouseDown 직전 `pyautogui.moveTo` 강제 호출 처리. 도형 유실이나 노이즈 발생 시 마우스가 엉뚱한 위치로 튀는 현상을 막기 위해 이전 좌표 대비 프레임 간 40픽셀 초과 이동 발생 시 노이즈로 보고 강제 기각(Outlier Rejection)하는 거리 기반 필터 탑재. 유실 시 가던 관성 속도 벡터의 크기를 최대 15픽셀로 제한(Clamping)하여 매끄럽고 안정적인 선형제어(Inertial Linear Drag)를 달성함.
- **v1.2.0 (2026.06.03)**: 타겟 초기 락온 및 트래킹 성능 극대화 패치. 팝업 감시 시작 후 2초 골든타임 이내에만 정가운데 부근(Center 240x240px 범위)에서 초기 흰색 도형을 락온하도록 제한. 2초 초과 시 락온 타임아웃으로 상태를 자동 리셋하여 오작동 방지. 매 프레임 파일 IO로 인한 CPU 병목을 bg_cache 메모리 캐싱으로 전면 개선. 타겟 정상 추적 시 `lost_cnt` 누적 오류를 해결하여 안정성 확보. 흰색 타겟 그레이스케일 임계치 완화(200 이상) 및 락온 탐색 윈도우 정밀화(120px).
- **v1.2.1 (2026.06.03)**: 도형 내부 격자 특징점(Dense Grid Features) 생성 기법 도입 및 코사인 유사도 궤적 필터 구현. OpenCV `goodFeaturesToTrack` 의 배경 무늬 엣지 오인식 문제를 해결하기 위해, 검출된 도형 윤곽선 내부 영역에 10px 간격의 바둑판식 격자점을 직접 특징점으로 매핑하여 타겟 도형만 뒤덮도록 조치. UI 데시보드 피드백 영상 상에 격자점들을 초록색 점으로 직접 렌더링하도록 구현. 트래킹 중 이전 이동 속도 벡터 대비 80도 이상의 급격한 방향 변경 발생 시 노이즈로 간주하고 해당 감지를 기각한 뒤 기존 관성 속도 벡터로 궤적을 보간 추적하는 코사인 유사도 각도 필터 추가.
- **v1.2.2 (2026.06.03)**: 실시간 동적 배경 캡처(DBBC) 도입 및 락온 정밀화 패치. 초기 타겟 도형이 흰색이 아닌 회색 또는 흐릿한 변색 상태일 때 흰색 임계치(white_mask) 의존성 때문에 배경 돌멩이 무늬를 타겟으로 오작동하는 락온 대실패를 근본 해결. 락온 탐색 진입 첫 프레임을 `dynamic_bg` baseline으로 박제하고 실시간 차분(`absdiff`)을 구하도록 개선하여 정지 상태인 돌무늬 노이즈는 100% 소거하고 정가운데서 새로 생성되는 타겟만 검출하도록 전면 개편. 락온 탐색 윈도우 한계를 팝업 중심 기준 반경 `80px`로 엄격히 좁혀 주변부 노이즈 유입을 차단.
- **v1.2.3 (2026.06.03)**: 고밀도 특징점(5px 격자) 도입 및 락온 2초 제한 해제 패치. 락온 기동 시의 2.0초 대기 한계를 전면 삭제하여 팝업 감지 상태에서 무제한 대기 락온이 가능하도록 조치. 초록색 특징점(격자점) 생성 간격을 기존 `10px` -> **`5px`**로 크게 축소(밀도 4배 상승)하여 타겟 도형 내부를 초록색 점들이 아예 면처럼 완전히 뒤덮고 추적하도록 극대화. 락온 탐지 마스크를 실시간 차분(`diff_mask` > 15)과 흰색 인식(`white_mask` > 180)의 합집합(`bitwise_or`)으로 개편하여 어두운 회색이나 미세 변색 도형의 초기 반응성 증대. EMA 진행 속도 벡터 및 관성 유지력 보완을 통해 궤적 튐 필터 기각 시 부드러운 테두리 추종 연속성 확보.
- **v1.2.4 (2026.06.04)**: 보안 감지 규칙 우회 패치. 키 이벤트의 윈도우 큐 중복 인젝션을 차단하기 위해 `self.key_states` 키 상태 추적 메모리 관리 기능을 탑재하여 `KeyUpEventRepeated` 차단. 단순 API 입력을 하드웨어 수준으로 숨기기 위해 `Windows SendInput API` 및 `Logitech G HUB virtual hardware driver` 연동 모듈을 구현하고 UI 시스템 환경 탭에 "입력 시뮬레이션 방식" 선택 옵션(PyAutoGUI, SendInput, Logitech G HUB)을 제공 및 프로필 저장 기능과 연동 완료.
