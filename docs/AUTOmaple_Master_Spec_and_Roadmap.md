# AUTOmaple Master Specification & Roadmap

- **배포 일자**: 2026.06.03
- **원본 백업 경로**: `deploy/backup/v1.0.0/`

## 2. 핵심 기술 사양 (v1.0.0)
### 2.1 Human-Engine (지능형 트래킹)
- **알고리즘**: 베지에 곡선(Bézier Curve) 및 동적 선형 보간(Lerp) 결합.
- **특징**:
  - 목표까지의 거리에 비례한 가속도/감속 제어.
  - 인간 특유의 오버슈트(Overshoot) 및 미세 손떨림(Micro-shaking) 시뮬레이션 적용.

### 2.2 도형 추적 시스템 (Computer Vision)
- **전처리**: `game_background_pattern.png` 기반의 정적 배경 차분(`cv2.absdiff`).
- **타겟 인식**: `anti1.png` 이미지에서 동적으로 추출한 타겟 색상(HSV) 마스킹.
- **최적화**: 모폴로지 연산을 통한 노이즈 제거 및 약 50FPS 수준의 실시간 처리 루프.

### 2.3 UI & 시스템 대시보드
- **데이터 바인딩**: `create_data_row` 메서드 구현을 통해 AttributeError를 해결하고 실시간 시스템 지표(가동 시간, 누적 사냥, 명령 횟수, 탐지 기록)를 QLabel에 출력.
- **안전 장치**: 전역 단축키(F2: 엔진 시작, F3: 엔진 중지) 및 비정상 종료 시 마우스 상태 강제 해제.

## 3. 차세대 기술 로드맵 (v1.0.1+)
사용자의 최우선 요청 사항인 **Lucas-Kanade 광학 흐름(Optical Flow)** 알고리즘을 도입하여 추적 성능을 극대화합니다.

### 3.1 Lucas-Kanade 알고리즘 도입 (진행 예정)
- **목표**: 기존 배경 차분/색상 기반 추적의 한계를 극복하고 픽셀 이동 벡터를 활용한 초정밀 추적 구현.
- **구현 방식**:
  - OpenCV의 `cv2.calcOpticalFlowPyrLK` 함수를 적용.
  - 투명 도형 내의 강한 특징점(Corners/Features)을 추출(예: `cv2.goodFeaturesToTrack`)하여 프레임 간 이동을 추적.
  - 배경이 복잡하거나 도형의 이동 궤적이 비선형적(당구공처럼 튀는 움직임)일 때 특징점 유실 없이 부드러운 드래그 상태를 유지.

## 4. 운영 및 버전 관리 원칙 (0.0.1 Rule)
1. **점진적 업데이트**: 모든 기능 추가 및 버그 수정 시 현재 버전에서 **0.0.1** 단위를 상향한다. (v1.0.0 -> v1.0.1 -> v1.0.2 ...)
2. **독립적 배포**: 각 업데이트 버전은 `deploy/dist/AUTOmaple_v1.0.x` 형태의 독립된 폴더로 빌드 및 배포한다.
3. **원본 보존**: v1.0.0은 정식 원본으로 정의하며, 어떠한 경우에도 백업본을 유지한다.
4. **문서 동기화**: 모든 기술적 결정 사항 및 수정 내역은 이 마스터 문서에 누락 없이 기록한다.
