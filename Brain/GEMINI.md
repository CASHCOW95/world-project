# LLM Wiki Management Rules (Schema)

이 파일은 지식/노하우 위키의 운영 원칙과 데이터 구조를 정의합니다. Gemini는 이 규칙에 따라 위키를 관리합니다.

## 1. 운영 체계

### [자료 넣기] (Ingestion)
- **사용자**: `raw/` 폴더에 새로운 자료를 추가합니다.
- **Gemini**: 
  1. `raw/` 폴더의 새로운 파일을 분석합니다.
  2. 설정된 미션에 따라 "핵심 Takeaway"를 도출합니다.
  3. **새로운 위키 페이지를 생성하며, 반드시 '💡 쉬운 설명'과 '🔍 원문 비교(표)' 형식을 포함합니다.**
  4. 생성된 페이지를 관련 MOC(`wiki/core/MOC - *.md`)에 링크로 연결합니다.
  5. `wiki/index.md`와 `wiki/log.md`를 업데이트합니다.

### [질문하기] (Query)
- **사용자**: 특정 주제에 대해 질문하거나 지식 간의 연결고리를 묻습니다.
- **Gemini**: `wiki/` 내의 정보를 바탕으로 답변하며, 관련 위키 페이지 링크를 함께 제공합니다.

### [건강검진] (Health Check)
- **Gemini**: 정기적으로 위키의 상태를 점검합니다.
  - 끊긴 링크(Broken Links) 확인
  - 중복된 지식의 통합
  - 카테고리 분류의 적절성 검토 및 `index.md` 재구성

## 2. 문서 규격

### 파일 구조
- 핵심 대시보드(MOC)는 `wiki/core/` 폴더에 관리합니다.
- 개별 지식 문서는 하위 폴더 없이 `wiki/` 폴더 내에 플랫하게 위치합니다.

### 위키 페이지 템플릿
```markdown
> [!IMPORTANT]
> **내 비즈니스/방향성 관점에서의 핵심 Takeaway**
> (사용자의 미션과 방향성을 고려한 이 내용의 핵심 가치 요약)

# {문서 제목}

## 💡 쉬운 설명 (Concept)
{어려운 개념을 누구나 이해할 수 있게 쉽게 풀어서 설명}

## 🔍 원문 비교 (Comparison)
| 핵심 포인트 | 쉬운 해설 | 원문 내용 (Reference) |
| :--- | :--- | :--- |
| 키워드/주제 | 풀어서 설명한 내용 | 원본 자료의 핵심 문구/데이터 |

## 🧠 심화 분석
{위키 링크를 활용한 추가 분석 및 연결}

## 연관 지식
- [[관련페이지1]]
- [[관련페이지2]]
```

## 3. 개인화 맥락 (User Context)

- **역할**: 바이브코딩, 코인트레이더, 부업
- **주요 관심사**:
  1. 경제지식 (시장 분석, 거시 경제, 재테크)
  2. AI 트렌드 (LLM, 자동화, 새로운 도구)
  3. 코드 엔지니어링 (Vibe Coding, 파이썬, 개발 효율화)
  4. 보컬 트레이닝 (발성, 노래 노하우, 연습 기록)
  5. 인생진리 (철학, 본질적 가치, 의사결정의 원칙, 깨달음)
- **비즈니스 미션/방향성**: 지식 축적을 통한 수익화 및 업무 효율화 극대화
